Authentication & Access

  • Valid user can log in successfully.
  • Invalid credentials are rejected with proper error messages.
  • Locked/deactivated users cannot log in.
  • User is successfully logged out.
  • Session is cleared, and no admin links are visible after logout.
  • Only staff members can access /dashboard.
  • Non-admins are redirected away from protected routes.
  • Attempting to access admin URLs without staff status triggers a 403 Forbidden.

User State Management

  • Activate/Deactivate user buttons trigger proper state changes.
  • Deactivated users are blocked from login.
  • Re-activated users regain access immediately.

Password Reset

  • Admin-triggered password reset sends an email and updates the password.
  • Password reset link in email is valid and leads to password update page.
  • Reset attempts on deactivated users are forbidden (403).
  • Reset attempts on nonexistent users return a 404.

CRUD Operations

  • Create, Edit, Delete boxes through the admin dashboard.
  • Orphaned products reassign properly when boxes are deleted.
  • Verify archiving status is triggered correctly.
  • Add, Edit, Remove products individually and within boxes.
  • Orphaned products appear in the dashboard correctly.
  • Reassign orphaned products works as intended.
  • Admin can edit user details (email, password, etc.).
  • Admin can toggle user state (activate/deactivate).
  • Admin can trigger password resets.
  • Create, Edit, and update order status.
  • Cancel subscription logic reflects accurately in user accounts.

Form Validation & Edge Cases

  • All form fields display correctly.
  • Required fields are marked and validated.
  • Errors are visible on the form when invalid data is entered.
  • CSRF token is included in all forms.
  • Handles excessively long inputs gracefully.
  • Invalid characters are sanitized.

Modal Interactions

  • Password Reset Modal opens correctly, triggers backend call.
  • Handles wrong password gracefully.
  • Displays proper error messages.
  • Toggle User State Modal reflects the correct state (Active/Inactive).
  • Disables/enables buttons correctly.
  • Redirects/reloads the admin list view on success.

Routing & URL Checks

  • Navigating to a non-existent user/product/box ID shows a 404 page.
  • Non-staff users accessing admin URLs show a 403 page.
  • Custom 404 page renders with proper styling.
  • Breadcrumbs are visible and clickable.
  • Navigation bar highlights the current page.

Security & Permissions

  • All forms include a valid CSRF token.
  • Submissions without CSRF token are rejected.
  • Non-admin users cannot see or interact with admin buttons.
  • Forbidden pages are gracefully handled (403 or redirect).

Front-end Validation

  • Button states, form field constraints, client-side validation in modals.
  • Button click events trigger without console errors.
  • Form constraints are respected (min/max length, special chars).
  • Select boxes and dropdowns are populated correctly.
